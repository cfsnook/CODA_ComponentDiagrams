8.1 Multiple Self Wake queues

In previous CODA versions, a component owned one implicit queue of wake events. This meant that it was not possible to distinguish between wake events sent for different (domain level) purposes. A component may now own several different wake queues, which are added by the user to represent different self-wake semantics (Figure 51). SelfWake operations must specify which of the component’s wake queues they wake on (Figure 52), and Operations that set a wake event must specify which wake queue the event is set in (Figure 53).
