8.4 Refactoring Menu: Commit and Revert

The Commit and Revert actions are accessed from a pop-up context menu, by right clicking on a machine that has outstanding uncommitted changes.

Figure 57 - Context menu for Commit/Revert.

Commit - When Commit is selected for a machine that has outstanding uncommitted changes, firstly an information and confirmation message appears. This gives the user a chance to cancel since the Commit operation is a significant process that can only be reversed by restoring from archive.

Figure 58 - Commit confirmation message. If ok is selected on Mi, the following steps will be taken

  1. any open diagrams will be closed,

  2. the project will be archived,

  3. the recorded changes for Mi will be propagated to the next level refinement Mi+1 (if any),

  4. the changes made to the refinement Mi+1 will be recorded,

  5. the previous change records for Mi will be deleted,

  6. all diagrams of the previous model Mi will be generated,

  7. steps 3 to 6 will be repeated for the refinement Mi+1,

  8. a report of the complete process will be saved.

Revert - When Revert is selected for a machine that has outstanding uncommitted changes, firstly an information message appears. This gives the user a chance to cancel since the Revert operation is a significant process that cannot be reversed.

Figure 59 - Revert confirmation message. If ok is selected on a machine Mi, the following steps will be taken

  1. any open diagrams will be closed,

  2. the changes made to Mi will be reverted taking the model back to its original state,

  3. the change records for Mi will be deleted,

  4. all diagrams of Mi will be generated.

No Changes - If Commit or Revert are selected for a machine that has no recorded changes, an appropriate message is displayed and no further action is taken.

Figure 60 - No changes error message.
